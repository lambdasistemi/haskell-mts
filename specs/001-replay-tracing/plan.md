# Implementation Plan: Replay Tracing

**Branch**: `001-replay-tracing` | **Date**: 2026-03-27 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-replay-tracing/spec.md`

## Summary

Extend the journal replay phase with entries-remaining progress
tracking across all three replay entry points: composed CSMT
(`mkKVOnlyOps`), standalone CSMT (`csmtReplayJournal`), and MPF
(`mpfReplayJournal`). The existing `ReplayEvent` type gains an
`rsEntriesRemaining` field, and the two standalone functions gain
a trace callback parameter.

## Technical Context

**Language/Version**: Haskell (GHC 9.8.4)
**Primary Dependencies**: rocksdb-kv-transactions, async, cereal
**Storage**: RocksDB (persistent), in-memory (testing)
**Testing**: Hspec + QuickCheck (hspec-discover)
**Target Platform**: x86_64-linux, aarch64-darwin
**Project Type**: Library (monorepo with sublibraries)
**Performance Goals**: Trace callback overhead negligible vs. I/O
**Constraints**: Callback invoked per chunk, not per entry
**Scale/Scope**: 3 functions modified, ~50-80 lines changed
